# Contributing to miniwing

Thanks for contributing.

## Development Setup

1. Fork and clone the repository.
2. Install dependencies:
   `npm install`
3. Run a local build:
   `npm run build`

## Project Structure

- `packages/miniwing/`: core CLI and generator logic
- `apps/docs/`: static documentation site
- `playground/`: manual testing playground

## Pull Request Guidelines

1. Create a branch from `main`.
2. Keep changes focused and atomic.
3. Update documentation if behavior changes.
4. Ensure CI passes before requesting review.

## Commit Message Guidelines

Use clear, imperative commit messages. Example:
- `fix(cli): resolve output path from repo root`
- `docs: add Netlify deployment section`

## Reporting Issues

Use the issue templates for:
- Bug reports
- Feature requests

## Code of Conduct

By participating, you agree to follow [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md).
